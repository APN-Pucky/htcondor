\subsection*{\label{sec:New-8-1-0}Version 8.1.0}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\noindent Release Notes:

\begin{itemize}

\item HTCondor version 8.1.0 not yet released.
%\item HTCondor version 8.1.0 released on Month Date, 2013.

\end{itemize}


\noindent New Features:

\begin{itemize}

\item Support for resource Consumption Policies on partitionable slots and 
the consumption of partitionable slot resources by the Negotiator 
during matchmaking, for slots advertising a Consumption Policy.
\Ticket{3435}

\item Added support for publishing information about an HTCondor pool to \TM{Ganglia}.
See page~\pageref{sec:Config-gangliad} for more information.
\Ticket{3515}

\item Improved performance of the \Condor{collector} when running
at sites that do not observer daylight savings times.
\Ticket{2898}

\item \Condor{q}, \Condor{rm}, \Condor{status} and \Condor{qedit} are now more consistent
in the way the handle \Opt{-constraint}.
\Ticket{1156}

\end{itemize}

\noindent Configuration Variable and ClassAd Attribute Additions and Changes:

\begin{itemize}

\item The new configuration variable
\Macro{FILE\_TRANSFER\_DISK\_LOAD\_THROTTLE} enables dynamic
adjustment of the level of file transfer concurrency in order to
keep the disk load generated by transfers below a specified level.
\Ticket{3613}

\item The following new \Condor{schedd} ClassAd attributes are for
monitoring file transfer activity:
\AdAttr{TransferQueueMBWaitingToDownload},
\AdAttr{TransferQueueMBWaitingToUpload}, and
\AdAttr{FileTransferDiskThrottleLevel}.
\Ticket{3613}

\item The default value for the configuration variable
\Macro{PASSWD\_CACHE\_REFRESH} has been changed from 300 seconds to
72000 seconds (20 hours).
\Ticket{3723}

\item Created the \Condor{dagman\_metrics\_reporter} executable to report metrics for
DAGMan jobs running under Pegasus
\Ticket{3532}

\end{itemize}

\noindent Bugs Fixed:

\begin{itemize}

\item Fixed a bug that could cause the daemons to crash on unix if the
OS reported that a job owner's account didn't exist (e.g. due to a
temporary NIS or LDAP failure).
\Ticket{3723}

\item Fixed a bug that resulted in a misleading error message when
\Condor{status} \Opt{-constraint} was given a constraint that could not
be parsed.
\Ticket{1319}

\item Fixed a typo in \Condor{q} output.
\Ticket{3703}

\end{itemize}
